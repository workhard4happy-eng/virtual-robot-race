# 用語集 (Glossary)

このドキュメントは、airaレッスンプロジェクトで使われる専門用語や重要なキーワードをまとめたものです。

---

### **A-Z**

*   **ACTIVE_ROBOTS**: `config.txt`内の設定項目。レースに参加するロボットの台数（1または2）を指定する。
*   **AIアシスタント (AI Assistant)**: Gemini Code Assist、Claude Code、Codexなど、コーディングを支援してくれるAI。このレッスンでは積極的に活用することが推奨されている。
*   **AIモード (AI Mode)**: ニューラルネットワークモデルによる推論結果に基づき、ロボットが自律走行するモード。ランチャーの **R mode=`ai`** で選択できる。
*   **aira**: "autonomous intelligence racing arena"の略。本仮想ロボットレースプラットフォームの名称。
*   **CNN (Convolutional Neural Network)**: 畳み込みニューラルネットワーク。画像データから特徴を抽出するのによく使われる。
*   **config.txt**: プロジェクト全体の設定ファイル。主に走行させるロボットの台数(`ACTIVE_ROBOTS`)などを指定する。
*   **CUDA**: NVIDIA製のGPUで並列計算を行うためのプラットフォーム。AIの学習や推論を高速化できる。
*   **DAgger (Dataset Aggregation)**: 模倣学習の一種。AI自身の走行データも学習データに追加していくことで、未知の状況への対応力を高める手法。
*   **DATA_SAVE**: `config.txt`内の設定項目。`1`にすると走行データ（画像、`metadata.csv`など）を保存する。`0`にすると保存しない。
*   **drive_torque**: 駆動輪に与えるトルク（回転させる力）を指示する変数。-1.0（最大後進）から+1.0（最大前進）までの値をとる。ロボット制御の基本出力の一つ。
*   **E2E (End-to-End)**: 「端から端まで」の意味。AIモードにおいて、画像の入力から操作の出力まで、すべてを単一のニューラルネットワークで処理する方式。
*   **Early stopping**: AIの学習において、検証用データに対する性能が改善しなくなった時点で学習を早期に打ち切る手法。過学習を防ぐ効果がある。
*   **Fork**: 他のユーザーのリポジトリを自分のアカウントにコピーするGitHubの機能。
*   **Git**: プログラムのソースコードなどの変更履歴を記録・追跡するためのバージョン管理システム。
*   **Google Colab**: Googleが提供する、WebブラウザからPythonのコードを実行できるサービス。GPUを無料で利用できるため、AIの学習時間を大幅に短縮できる。
*   **GPU (Graphics Processing Unit)**: 画像処理を担当する半導体チップ。AIの学習で必要となる大量の計算を高速に処理できる。
*   **loss (損失)**: AIの学習において、モデルの予測結果と正解データとのズレ（誤差）を表す指標。学習はこのlossを最小化することを目的とする。
*   **metadata.csv**: 走行ログの中心となる最も重要なデータファイル。`tick`ごとの全ロボット状態（操作、位置、SOCなど）が記録されている。
*   **MIMO (Multiple Input, Multiple Output)**: 複数の入力から複数の出力を決定する制御システム。このプロジェクトでは「画像とSOC」を入力とし、「アクセルとハンドル」を出力する。
*   **MODE_NUM**: `config.txt`内の設定項目。ロボットの制御モードを指定する。`1`:キーボード, `2`:テーブル, `3`:ルールベース, `4`:AI, `5`:スマートフォン。ランチャーの **R1 mode / R2 mode** ドロップダウンで設定できる。
*   **NAME**: `config.txt`内の設定項目。リーダーボードに表示されるプレイヤー名。
*   **PyTorch**: オープンソースの機械学習ライブラリ。主にAIモデルの学習と推論に使用される。
*   **RACE_FLAG**: `config.txt`内の設定項目。`1`にすると走行結果がリーダーボードに登録される。`0`はテスト走行のみ。
*   **SOC (State of Charge)**: バッテリーの充電残量。1.0が満充電、0.0が空の状態を表す。
*   **steer_angle**: 前輪の操舵角を指示する変数。単位はラジアン。ロボット制御の基本出力の一つ。
*   **table_input.csv**: テーブルモードで使用されるCSVファイル。`time_id`ごとの`drive_torque`と`steer_angle`が記述されており、この通りにロボットが走行する。
*   **tick**: シミュレーション内の時間の最小単位。このプロジェクトでは20fps（50ms）で1tickが進む。
*   **Unity**: 3Dおよび2Dのビデオゲームやインタラクティブコンテンツを作成するためのクロスプラットフォームゲームエンジン。このプロジェクトではシミュレーターとして機能する。
*   **Upstream**: Gitにおいて、自分がフォークしてきた元の（本家の）リポジトリを指すリモート名。本家の更新を取り込む際に使う。
*   **Vibe Coding**: AIアシスタントと対話しながら、まるで会話やセッションのようにプログラミングを進めていく開発スタイル。明確な指示を出すだけでなく、アイデアの壁打ち相手にすることなども含む。
*   **VSCode (Visual Studio Code)**: Microsoftによって開発されたソースコードエディタ。

---

### **あ-ん**

*   **アーキテクチャ (Architecture)**: システム全体の構造や設計思想。
*   **逆走 (Wrong-way driving)**: コースを通常とは反対の方向に走ること。ペナルティとして周回数が減る。
*   **エポック (Epoch)**: AIの学習において、データセット全体を1回すべて学習させるサイクルの単位。
*   **オフライン学習 (Offline Learning)**: 走行中にリアルタイムで学習するのではなく、事前に収集した走行ログデータ（バッチデータ）を使って学習を行う方式。
*   **ゲイン (Gain)**: 制御工学において、入力の変化に対して出力がどれだけ変化するかの比率。
*   **強化学習 (Reinforcement Learning)**: ロボットが試行錯誤しながら、より良い結果（報酬）を得られるような行動を学習する手法。(**注**: 本プロジェクトの`rl_reward.py`は、リアルタイム学習ではなく走行後のデータ品質を評価（スコアリング）するためにこの概念を応用している。)
*   **グリッド (Grid)**: レースのスタート位置。
*   **クローン (Clone)**: リポジトリを自分のPCにダウンロードしてくること。
*   **コースアウト (Course Out)**: ロボットがコースから外れて落下すること。失格となる。
*   **シーケンス制御 (Sequence Control)**: あらかじめ定められた順序や手順に従って、制御の各段階を逐次進めていく制御方式。テーブルモードがこれにあたる。
*   **スライディングウィンドウ法 (Sliding Window)**: 画像処理の手法の一つ。小さなウィンドウ（窓）を画像上でスライドさせながら、各部分領域の情報を解析する。白線検出などで使用される。
*   **推論 (Inference)**: 学習済みのAIモデルを使って、新しいデータに対する予測や判断を行うこと。
*   **タイムアップ (Time Up)**: 規定時間内（90秒）に2周を完走できないこと。失格となる。
*   **テンソル (Tensor)**: 多次元配列。PyTorchなどのAIフレームワークでは、数値データを扱うための基本単位として使われる。
*   **トルク (Torque)**: タイヤを回転させる力。`drive_torque`として制御する。
*   **ニューラルネットワーク (Neural Network)**: 人間の脳の神経回路網を模倣した数理モデル。AIの頭脳にあたる部分。
*   **ハイブリッドモード (Hybrid Mode)**: スタート検出など確実性が求められる部分はルールベースで、走行制御はAIで行うなど、複数の方式を組み合わせた制御モード。
*   **フィードバック制御 (Feedback Control)**: センサーなどで現在の状態を観測し、目標とのズレを修正するように制御する方式。ルールベース制御やAI制御がこれにあたる。
*   **フライング (False Start)**: レースのスタート合図より前に動き出してしまう反則行為。
*   **分布シフト (Distribution Shift)**: AIの学習時に使われたデータの分布と、実際に使用される（推論時の）データの分布が異なること。模倣学習では、人間が走行したことのない状況に遭遇するとこれが原因でうまく動けなくなることがある。
*   **模倣学習 (Imitation Learning)**: 人間の運転データ（お手本）をAIに学習させ、その操作を真似させることで自律走行を実現する手法。
*   **ラジアン (Radian)**: 角度の単位。`steer_angle`で使用される。 (πラジアン = 180度)
*   **ルールベース制御 (Rule-Based Control)**: 「もし白線が右にあれば左にハンドルを切る」のように、人間が定義した「ルール（if-then）」に基づいてロボットを制御する方式。ランチャーの **R mode=`rule_based`** で選択できる。
*   **ログデータ (Log Data)**: 走行中に記録された各種データ（画像、センサー値、操作量など）の集まり。`metadata.csv`が中心的な役割を担う。
*   **ロバスト (Robust)**: 外部からの予期せぬ影響（外乱）や環境の変化があっても、安定して性能を維持できる性質。堅牢性。
