# 2. NotebookLMを活用したライブQ&A

## はじめに
開発を進める中で、エラーに直面したり、コードの構造が分からなくなったりすることがあるでしょう。
そんな時、あなたの強力なサポーターとなるのが NotebookLM によるライブ QA システム です。

このツールは、aira の仕様書や実際のソースコードをすべて把握した「AI メンター」です。

> **▶ このレッスンをまず動画で見る**: [Lesson 02 NotebookLM活用法（YouTube）](https://youtu.be/kwLAPdCD-jU?t=150)

> **❓ 「普通のAIチャット（ChatGPT・Gemini等）ではダメなの？」**
> 通常のAIチャットは一般的な知識で答えますが、airaのソースコードや仕様書は学習していません。NotebookLMは「airaのリポジトリ・ドキュメント一式」を**ソースとして読み込んだ状態**で動作するため、「この関数はどこで定義されているか」「このエラーはairaのどの処理で起きているか」といったプロジェクト固有の質問に正確に答えられます。

## 2.1 ライブ QA へのアクセス方法
以下のリンクから、本レッスン専用のチャットウィンドウを開くことができます。

[NotebookLMで質問する](https://notebooklm.google.com/notebook/ab916e69-f78b-47c3-9982-a5210a07d713)

ログイン: ご自身の Google アカウントでログインしてください。

画面構成: チャット入力欄のみが表示されます。ソースコードやメモにアクセスすることはできませんが、対話を通じて必要な情報を引き出すことができます。

**まずはここで試してみましょう！** 以下のプロンプトをそのままコピーして送信してみてください。

```
このレースで勝つためのポイントは何ですか？
```

NotebookLM がどんな回答をするか確認し、AIメンターの雰囲気をつかんでおきましょう。

> **❓ 「自分の質問や自分のアルゴリズムは他の参加者に見えますか？」**
> **見えません。** チャット履歴はあなたのGoogleアカウントに紐づいた**個人のセッション**です。同じリンクを使っていても、他の参加者や運営側があなたの会話を閲覧することはできません。ただし、チャットで貼り付けた内容はGoogleのサービス利用規約（プライバシーポリシー）の対象となります。競技上の重要なアルゴリズムが気になる場合は、該当箇所だけを抜粋して質問するとよいでしょう。

## 2.2 NotebookLM で「できること」
この AI メンターには、以下のような質問を投げかけることができます。

- エラーの解決策を相談する 
  - 「python main.py を実行したら〇〇というエラーが出た。どう直せばいい？」

- コードの解説を求める 
  - 「main.py の中で Unity との通信（WebSocket）を処理しているのはどこ？」

- アーキテクチャの確認 
  - 「Rule-Based モードと AI モードで、意思決定ロジックはどう切り替わっている？」

- 仕様の確認 
  - 「SOC（バッテリー残量）がゼロになるとロボットはどうなる？」

> **❓ 「AIコーディングアシスタントとNotebookLMはどう使い分けるの？」**
> 役割が異なります。
> - **AIコーディングアシスタント（Gemini Code Assist、Claude Code、Codexなど、VS Code内）**: 今開いているファイルを見ながら、コードを書く・直す・補完するのが得意。「このエラーを修正して」「この関数を実装して」はこちら。
> - **NotebookLM（ブラウザ）**: airaの公式ドキュメントとリポジトリの初期状態を「辞書」として持ち、仕様・アーキテクチャの質問に答えるのが得意。「この仕組みはどう動くか」「どのファイルを修正すればよいか」はこちら。
>
> 「コードを書く道具」と「airaの辞書」として並行して使うのが最も効率的です。

## 2.3 良い回答を引き出すための「質問のコツ」
AI からより正確なアドバイスをもらうために、以下の情報を添えて質問することをお勧めします。 

- 「何」をしようとして: （例：マニュアルモードで動かそうとして）
- 「どんな」エラーが出たか: （例：ターミナルに表示されたエラー文をコピペ）
- 「どこまで」確認したか: （例：00番のセットアップ手順は完了している）
- 「自分が変更したコード」: 変更した箇所をコピペして渡す（下記参照）

NG例: 「動きません。助けてください。」
OK例: 「main.py を実行しましたが、Unity 画面が真っ暗なままです。ターミナルには ConnectionRefusedError と出ています。確認すべきポイントはどこですか？」

> **❓ 「NotebookLMは私が書き換えたローカルのコードも把握しているの？」**
> **いいえ。** NotebookLMが知っているのは、**公式リポジトリの初期状態**のみです。あなたがローカルで変更した `rule_based_algorithms/` のコードや最新のパラメータは見えていません。自分が書いたコードについて質問するときは、**該当箇所をチャットにコピペして**渡してください。「このコードを見てください：[コードを貼り付け]。なぜこの挙動になるのでしょうか？」という形が有効です。

> **❓ 「AIが間違ったことを言った場合、どう見破ればいい？」**
> NotebookLMはソースに基づいているため一般的なAIより正確ですが、存在しない変数名や関数を教えることがあります。**確認方法**: 「このファイルのこの関数」と教えられたら、VS Codeで `Ctrl+Shift+F` を使ってプロジェクト全体を検索してみてください。見つからなければ回答を疑い、「それは見当たりませんでした。もう一度確認してください」と再質問してみましょう。

## 2.4 ワークフローへの組み込み
レッスン中は、VS Code でコードを書きながら、横でブラウザの NotebookLM を開いておきましょう。
「ちょっと確認したい」という時に即座に解決できるため、学習効率が飛躍的に高まります。
