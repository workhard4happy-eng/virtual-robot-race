# 3. マニュアル操作

ここでは、キーボードでロボットを手動操作し、コースを走り切る体験をします。
自分で操作することで、「速く走るとはどういうことか」「なぜミスが起きるのか」をつかんでおきましょう。これが、後のレッスンでアルゴリズムを設計するときの「直感」になります。

> **▶ このレッスンをまず動画で見る**: [Lesson 03 マニュアル操作（YouTube）](https://youtu.be/kwLAPdCD-jU?t=197)

---

## 1. 起動と設定

### ランチャーで設定する

`start.bat` を実行するとランチャーが開きます。以下のように設定してください。

| 項目 | 設定値 | 説明 |
|------|--------|------|
| **Name** | あなたの名前 | 英数字とアンダースコア、最大16文字 |
| **Competition** | Tutorial | そのままでOK |
| **Active** | `1` | Robot1だけ使用 |
| **R1 mode** | `keyboard` | キーボード操作モード |
| **Data save** | `ON` | 走行データを保存する（後のレッスンで使用） |
| **Race flag** | `TEST ONLY` | 練習なのでリーダーボードには送らない |

設定を確認したら **START** をクリックします。Unityが自動で起動し、レースが始まります。

> **💡 config.txt を直接編集したい場合**: ランチャーで設定した内容は `config.txt` に自動で書き込まれます。上級者はファイルを直接編集してもかまいません。ただし、デフォルトの `HEADLESS=0` のままではランチャーGUIが開き、**STARTボタンを押した時点のGUI設定が優先されて上書き**されます。GUIを経由せずにconfig.txtの設定をそのまま使いたい場合は、`HEADLESS=1` に設定してから起動してください。

## 2. 基本ルール

> **▶ 動画チャプター**: [基本ルールの解説](https://youtu.be/kwLAPdCD-jU?t=197)

- **スタート:** 赤いシグナルが3つ点灯し、その後すべて消灯します。消灯がスタートの合図です。
- **フライング:** スタート合図の前に動くとフライングとなり、失格になります。
- **周回:** コースを2周し、その合計タイムを競います。
- **逆走:** コースを逆走すると、周回数がマイナス1されてしまいます。
- **バッテリー (SOC):** 走行するとバッテリーを消費します。SOCが0%になると走行不能になります。
- **衝突:** 壁や他のロボットに衝突すると、ペナルティとしてSOCが減少します。
- **コースアウト:** コースから落下すると失格です。
- **タイムアップ:** 90秒以内に2周を完了できない場合も、タイムアップで失格となります。

## 3. キーボード操作

| キー | 操作 |
|------|------|
| `W` | 前進 |
| `Z` | 後退 |
| `J` | 左ステアリング |
| `L` | 右ステアリング |
| `I` / `M` | ステアリング中央（即座に戻す） |

> `J` / `L` キーを離すと、ステアリングは自動で中央に戻ります。`I` / `M` で即座に中央に戻すこともできます。

## 4. 画面の見方

### カメラ映像
2台起動している場合、左側がRobot1、右側がRobot2のカメラ映像です。

### ターゲット表示
画面上の丸い円は「ターゲット」です。キーボード入力（`W`, `Z`, `J`, `L`）をXY座標で可視化したものです。前進トルク最大・ステアリング中央なら座標 (0, 1) に移動します。

> **❓ 「このXY座標って何を表しているの？」**
> X軸 = ステアリング角（-0.524〜+0.524 rad）、Y軸 = ドライブトルク（-1〜+1）で、実際にUnityへ送信される**連続値の制御信号**をそのまま可視化しています。キーを押すたびに値がどう変化するか、円の動きで確認できます。

### ロボット背面のランプ
後続ロボットへ自分の操作を伝える視覚的なインターフェースです。

| 表示 | 意味 |
|------|------|
| **色（Hue）** | ステアリング方向：赤=左、緑=直進、青=右 |
| **光の高さ** | 前進アクセルの強さ（下から上へ伸びる） |
| **点滅** | バック（後退）中 |

> **❓ 「テールランプの色はAIの視覚特徴量として意図されたもの？」**
> はい、意図的な設計です。後続ロボットのカメラ画像には前のロボットのテールランプが映り込むため、AIが「前車の色（ステアリング方向）」を画像特徴として学習できます。このページ下部の課題4で、AIのテールランプを観察してみてください。色の変化がAIの意思決定のリアルタイムな可視化になっています。

## 5. レッスン課題

毎回 `start.bat` を起動してランチャーで設定を変更してから **START** してください。

> **❓ 「キーボード入力はBang-Bang制御になってAI学習データとして使い物になるの？」**
> 厳密なBang-Bangではありません。キーを**押し続けると**トルク・ステアリングが50msごとに段階的に増加し（`TORQUE_STEP=0.25`、`STEER_STEP=0.20`）、**離すと即座に0**にスナップします。ローパスフィルタは入っていません。そのため、**キーを「チョンチョン」と短く叩く**ことが滑らかなデータを作るコツです。ガクガクした粗い操作を保存すると、Lesson 06でAIも同じ粗い走りを学習します。

> **❓ 「通信ラグ（50ms）は学習データに悪影響を与えないの？」**
> **問題ありません。** WebSocketの50ms周期は一定のため、操作から反映までのラグは常に一定です。保存されるデータは「あなたが送った制御コマンド」と「Unityから受け取ったセンサー」が同じ周期で記録されます。AIの推論実行時も同じ50ms周期で動くため、ラグは学習・推論の両方で対称的にかかり、実質的な問題になりません。

### 課題1: Robot1で2周完走を目指す
ランチャー設定: **Active=`1`、R1 mode=`keyboard`、Data save=`ON`**

2周完走するまで何度でも挑戦しましょう。

### 課題2: Robot2で2周完走を目指す
ランチャー設定: **Active=`2`、R2 mode=`keyboard`**

Robot2に切り替えて同じコースを走ります。操作感の違いに気づきますか？

### 課題3: 失敗を体験する
フライング、コースアウト、衝突によるバッテリー切れなど、様々な失格パターンを**意図的に**試してみましょう。ペナルティの仕組みを体で覚えます。

### 課題4: AIと併走する
ランチャー設定: **Active=`1,2`、R1 mode=`keyboard`、R2 mode=`ai`**

AIの走りを観察しながら追走してみましょう。どこで差がつくか確認してみてください。

### 課題5: タイムアタック
ランチャー設定: **Active=`1`、R1 mode=`keyboard`**

2周の合計タイムを競いましょう。自己ベストを更新してみてください！
